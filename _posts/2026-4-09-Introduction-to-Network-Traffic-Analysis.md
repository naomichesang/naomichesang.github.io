Network Traffic Analysis (NTA) is the practice of examining network traffic to understand ports and protocols, establish 
baselines, detect anomalies, and respond to threats. It gives security teams visibility into the 
network, helps meet compliance standards, and counters attackers who often use legitimate tools and 
credentials to evade detection.

Common tools used in NTA include: *Wireshark*, *TCPDump*, *Tshark*. 

## Wireshark
Wireshark is a premier, open-source network traffic analyzer (NTA) tool used for deep packet inspection, troubleshooting, and security analysis by capturing and interpreting live network data. It enables users to visualize traffic, filter by protocols, and analyze PCAP files to detect malicious activity or performance issues.

## TCPDump
Tcpdump is a powerful, command-line packet analyzer used in Network Traffic Analysis (NTA) for capturing, filtering, and analyzing network traffic in real-time on Unix-like systems. It is ideal for troubleshooting network latency, dropped packets, and security monitoring, providing detailed, raw packet data to identify anomalies or unauthorized traffic.

## Tshark
TShark is the command-line network traffic analyzer, acting as the terminal-based counterpart to Wireshark, essential for Network Traffic Analysis (NTA). It captures live data and reads .pcap files, allowing for deep packet inspection, protocol decoding, and filtering without a GUI, making it ideal for automation, scripting, and remote server analysis.
